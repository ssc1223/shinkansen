翻譯快速鍵

三組可自訂的翻譯預設,對應三個快速鍵。依網頁內容選擇不同引擎或模型（例如文學性強用最強模型、一般網頁用最省錢）。 已翻譯狀態下按任一快速鍵都會還原原文;翻譯中按任一快速鍵則取消翻譯。 鍵位可至 chrome://extensions/shortcuts 變更

⚠ Firefox 使用者注意:Firefox 的 Alt+S 會展開「歷史」選單、 Alt+D 會切到地址列，Alt+A 也常被其他擴充功能（例如 Save Page WE）攔截。 Chrome 沒這個衝突，但 Firefox 環境下三組預設快捷鍵可能無法觸發翻譯。 請到 about:addons → 設定圖示（齒輪）→「管理擴充功能快捷鍵」改成不衝突的組合（例如 Ctrl+Shift+S）

模型計價

Shinkansen 內建已知 Gemini 模型的標準層級單價（2026-05 校準）， 用於計算翻譯成本顯示在翻譯完成通知與「用量紀錄」分頁。Google 改價時可在下方覆蓋對應模型，空白表示用內建價。 詳細費率見 Gemini 官方定價頁

Preset 翻譯的計價優先順序：覆蓋值 → 內建表 → 下方「後備路徑單價」。單位皆為「每 1M tokens 的美金價格」

Flash Lite gemini-3.1-flash-lite 內建 $0.25 / $1.50
Flash gemini-3-flash-preview 內建 $0.50 / $3.00
Flash 3.5 gemini-3.5-flash 內建 $1.50 / $9.00

三欄依序為 input / output 單價(USD/M tokens)與 cache 命中折扣(%，例：90 = 命中部分省 90%)。 Gemini 2.5 起 implicit cache 預設 90% off

連本機 Ollama / llama.cpp 收到 403？（展開查看解法）

Chrome 從擴充功能呼叫本機後端（localhost / 127.0.0.1）時， 會強制送 Origin: chrome-extension://<id> header。 Ollama 與 llama.cpp 預設的 CORS 設定不認得這個 origin → 直接回 403

兩種解法擇一：

  1. Ollama：啟動前設環境變數 OLLAMA_ORIGINS=* （或更嚴格的 chrome-extension://*）。 macOS 用 launchctl setenv OLLAMA_ORIGINS "*" 後重啟 Ollama； Windows 在「系統內容 → 環境變數」加 OLLAMA_ORIGINS = *
  2. 後端不能改 CORS 設定時：在系統 hosts 檔加一筆 127.0.0.1 ollama.local，Base URL 改成 http://ollama.local:11434/v1。Chrome 看到的不是 loopback 地址就不會強制送 Origin
進階：思考強度 / 段序號 / 自訂 body 本機量化模型（如 gemma-4 量化版）會把預設的「«1» «2»」段序號誤譯為「N1、N2」洩漏到譯文。 開啟後改用 <<<SHINKANSEN_SEG-N>>> 格式，弱模型不會誤翻； 代價是每段批次多約 7 tokens（input + output 雙倍開銷）。 商用 API（OpenRouter / Groq 等）不需此選項，但開啟也無害

模型計價（USD）

請填入 input / output 單價， 系統會用來估算翻譯費用顯示在翻譯完成通知與「用量紀錄」分頁。填 0 代表不顯示費用 （token 數仍會記錄）

禁用詞清單

針對 AI 模型容易漏網的禁用辭彙自訂禁用對照表。內容會以高顯著性區塊 注入翻譯引擎的翻譯 Prompt 末端，明確要求譯文不可使用左欄詞彙、 必須改用右欄。修改清單後會自動讓既有翻譯快取分區，不再命中舊譯文
